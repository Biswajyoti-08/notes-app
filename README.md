# Notes API

A multi-user notes service with JWT authentication, note sharing, version history, full-text search, and pagination.

## Tech Stack

| Concern | Choice | Reason |
|---|---|---|
| Framework | FastAPI | Native async, auto OpenAPI, Pydantic validation |
| ORM | SQLAlchemy 2 | Battle-tested, supports SQLite → Postgres migration |
| Database | SQLite (local) / Postgres (prod) | Zero config locally; swap via `DATABASE_URL` env var |
| Auth | JWT via `python-jose` + `bcrypt` | Industry standard; stateless tokens |
| Deployment | Render.com | Free tier, `render.yaml` config included |

## Endpoints

### Auth
| Method | Path | Auth | Description |
|---|---|---|---|
| POST | `/register` | ✗ | Register new user |
| POST | `/login` | ✗ | Get JWT token |

### Notes (all require `Authorization: Bearer <token>`)
| Method | Path | Description |
|---|---|---|
| GET | `/notes` | All accessible notes (owned + shared). Supports `?page=&per_page=` |
| POST | `/notes` | Create a note |
| GET | `/notes/{id}` | Get one note |
| PUT | `/notes/{id}` | Update note (saves a version snapshot first) |
| DELETE | `/notes/{id}` | Delete note |
| POST | `/notes/{id}/share` | Share with another user by email |

### Custom Features
| Method | Path | Description |
|---|---|---|
| GET | `/notes/{id}/versions` | 📚 List all saved versions of a note |
| POST | `/notes/{id}/versions/{version_id}/restore` | ♻️ Restore note to a previous version |
| PATCH | `/notes/{id}/pin` | 📌 Toggle pin — pinned notes appear first |

### Stretch Goals
| Method | Path | Description |
|---|---|---|
| GET | `/search?q=keyword` | Full-text search across title + content |
| GET | `/openapi.json` | OpenAPI 3.0 spec (auto-generated by FastAPI) |
| GET | `/docs` | Swagger UI |
| GET | `/about` | API metadata |

## Custom Feature: Note Version History

Every time a note is updated via `PUT /notes/{id}`, the **previous title and content are automatically snapshotted** before the update is applied. Users can:

1. **Browse history** — `GET /notes/{id}/versions` returns all snapshots, newest first.
2. **Restore any version** — `POST /notes/{id}/versions/{version_id}/restore` rolls the note back. The current state is snapshotted first so nothing is ever permanently lost.

**Why?** Accidental overwrites are the #1 data-loss scenario in notes apps. Google Docs and Notion both offer version history for exactly this reason. It's a meaningful safety net with minimal implementation complexity.

## Running Locally

```bash
git clone <your-repo>
cd notes-app

python -m venv venv && source venv/bin/activate
pip install -r requirements.txt

cp .env.example .env          # edit SECRET_KEY
uvicorn main:app --reload
```

Visit `http://localhost:8000/docs` for the interactive Swagger UI.

---

## Deploying to Render.com

1. Push this repo to GitHub.
2. Go to [render.com](https://render.com) → **New Web Service** → connect your repo.
3. Render auto-detects `render.yaml`. Click **Deploy**.
4. Set the `SECRET_KEY` environment variable to a long random string in the Render dashboard.

The app uses SQLite by default. For production persistence, add a Render **PostgreSQL** database and set the `DATABASE_URL` environment variable to the connection string Render provides.

## Edge Cases Handled

- Duplicate email registration → 409 Conflict
- Wrong password → 401 with `{"detail": "Invalid email or password"}`
- Accessing another user's note → 403 Forbidden
- Sharing note with self → 400 Bad Request
- Sharing already-shared note → 409 Conflict
- Updating/deleting a note you don't own (shared) → 403
- Empty title → 422 Validation Error
- Missing or invalid JWT → 401
- Note not found → 404
- Password too short (< 8 chars) → 422 Validation Error
